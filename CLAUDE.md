## 1. プロジェクト概要

**何を作るか**: 選手が練習動画をアップロードすると、AIが走力・ボールコントロール等を分析し、独自スコアを算出。スカウト向けDBと選手育成機能を一体化したモバイルアプリ。

**ターゲットユーザー**:

- 選手（10代〜プロ志望）
- スカウト・エージェント
- コーチ

## 2. 技術スタック

| レイヤー         | 技術                                | 備考                                 |
| ---------------- | ----------------------------------- | ------------------------------------ |
| フロントエンド   | React Native + Expo                 | iOS / Android クロスプラットフォーム |

## 3. 担当責務

## 4. 自己検証ルール

実装・編集を終えたら、必ず以下を実行し結果を報告してから完了とする。

```bash
# バックエンド
cd backend && python -m pytest tests/ -v

# フロントエンド
cd mobile && npm test

# 型チェック
npx tsc --noEmit

# Lint
npm run lint
```

- 検証が通らない場合は修正してから再実行すること
- 人間に引き渡す前に自己完結させること

---

## 5. コーディング規約

- **関数は30行以内**（超える場合は分割を検討）
- **テストなしのコードはコミット禁止**（ユニットテスト or 統合テスト必須）
- **エラー時は必ず「原因・影響範囲・対応策」の3点を報告**
- **型安全を最優先**（TypeScript strict mode、Python type hints 必須）
- **AI関連の重い処理は非同期キュー経由**（ユーザーを待たせない設計）
- 個人情報（選手データ）は暗号化して保存すること

---

## 6. コミットメッセージ規約

```
feat: 新機能追加
fix: バグ修正
docs: ドキュメント更新
refactor: リファクタリング
test: テスト追加・修正
chore: 依存関係・設定変更
ai: AI/MLモデル・推論関連
```

例: `feat: 動画アップロード → AI分析キュー投入フローを実装`

---

## 7. 禁止事項・注意事項

- 選手の個人情報・動画を外部APIに無断送信しない
- 未成年選手データは保護者同意フラグを必ず確認してから処理
- AI分析結果を「確定的な評価」として提示しない（「参考スコア」として表示）
- スカウト向けDB は認証済みユーザーのみアクセス可能にする
