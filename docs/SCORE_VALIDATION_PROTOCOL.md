# スコア精度 検証プロトコル（A #1 / #4）

> **作成日**: 2026-07-13
> **目的**: AIスコアが「どれくらい信用できるか」を第三者に説明できる形で測る手順を定義する。
> 外販の生命線であり、これが無いと B2B 商談で「精度が信用できない」で止まる。

## 1. 何を検証するか

| 検証項目       | 問い                               | 指標                                          |
| -------------- | ---------------------------------- | --------------------------------------------- |
| **妥当性(#1)** | AIスコアは人の評価と一致するか     | 相関係数 r / 決定係数 R² / 順位一致(Spearman) |
| **再現性(#4)** | 同じ選手を測ると同じ結果か         | 同一動画の再測定の標準偏差 / 級内相関 ICC     |
| **頑健性(#3)** | 撮影条件が変わるとどれだけブレるか | 角度・距離・fps 別の平均絶対誤差(MAE)         |

## 2. 妥当性検証(#1) の手順

1. **サンプル収集**: 同一選手の動画を 30〜50 本（年代・ポジションを分散）。
2. **正解ラベル**: 資格を持つコーチ 2〜3 名が各動画を 5 項目 0-100 で独立採点。
3. **コーチ間信頼性**: コーチ同士の一致度(ICC)を先に確認（低ければ基準を再定義）。
4. **突合**: AIスコア vs コーチ平均を項目別に散布図化、r / R² / Spearman を算出。
5. **合格基準（暫定）**: 総合スコアで r ≥ 0.7、各項目で r ≥ 0.6 を目標。
6. **キャリブレーション**: ずれがあれば `reliability.py` の誤差レンジ・重みを更新。

## 3. 再現性検証(#4) の手順

1. 同一動画を 5 回解析 → 各スコアの標準偏差を算出（理想は 0 に近い）。
2. 同一選手の別テイク（同条件）を複数解析 → 級内相関 ICC ≥ 0.8 を目標。
3. 乱数・非決定的処理があれば固定 or 平均化。

## 4. 頑健性検証(#3) の手順

- 撮影条件マトリクス（角度: 正面/斜め/横 × 距離: 近/中/遠 × fps: 30/60）で同一プレーを撮影。
- 基準条件（正面・中・60fps）との MAE を条件別に集計。
- 影響の大きい条件を撮影ガイドライン(#44)に反映。

## 5. 公開物（営業で使える形）

- 「精度レポート」1枚: r / R² / ICC / 条件別 MAE、サンプル数、限界の明記。
- スコアUIの誤差レンジ(±)はこの検証結果でキャリブレーションする（実装済 A#2）。
