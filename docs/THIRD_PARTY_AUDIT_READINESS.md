## 1. 監査の狙い

外販の最終関門は「本当にその精度・公平性を、第三者が確認できるか」。
自己申告ではなく、**再現可能な証跡**と**独立評価の受け入れ体制**を用意する。

## 3. 監査準備チェックリスト

### データ・証跡

- [ ] 匿名化した監査用データセットを用意（未成年は同意済みのみ）
- [ ] スコア算出の入力→出力を再現できるログ／バージョン管理
- [ ] モデル・重み（position_weights 等）のバージョン固定と履歴

### 精度・公平性

- [ ] 相関検証（AI vs 専門家）の結果（決定係数）— A#1
- [ ] 再現性（テスト-再テスト）の結果 — A#4
- [ ] バイアス監査レポートの定期出力・閾値超過時の是正記録 — A#5
- [ ] 誤判定申告の受付〜補正の運用記録 — A#9

### ガバナンス

- [ ] スコアが「参考値」であることの明示（UI/契約/レポート）
- [ ] 誤差・限界の開示方針
- [ ] 情報セキュリティ体制（アクセス制御・削除/開示対応）— D#33
- [ ] 認証取得計画（Pマーク/ISMS）— D#34

---

## 4. 監査の進め方

1. **内部監査**: 上記チェックリストで自己点検 → ギャップを是正。
2. **第三者レビュー**: 研究機関(A#6)や外部監査法人にデータ・手順を開示。
3. **認証取得**: 情報セキュリティは Pマーク/ISMS、精度は学術的評価で担保。
4. **継続監査**: バイアス監査を定期実行し、閾値超過は是正して記録を残す。

---

## 5. 現時点のサマリ（総評）

- **技術的な監査材料（誤差・根拠・バイアス・補正・基準）はコードで実装済み**。
  第三者が「何を根拠にスコアが出て、どこに偏りがあるか」を確認できる状態にある。
- 残るは **実データによる相関/再現性の統計的証明（A#1/#4）** と
  **情報セキュリティ認証（D#34）**。この2つが外販の信頼性の最後のピース。
