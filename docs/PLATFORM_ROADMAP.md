## G#46 分析パイプライン安定化（リトライ・DLQ・SLA）— 一部実装済み

### 実装済み

| 対策                    | 実装                                                                |
| ----------------------- | ------------------------------------------------------------------- |
| タスクリトライ          | `analyze_video`: 最大3回・30秒バックオフ → 失敗時 FAILED＋通知      |
| 冪等性                  | 再実行時は既存 AnalysisResult を置換。COMPLETED はスキップ          |
| broker断への耐性        | `dispatch_analysis` は投入失敗でも API を落とさない（PENDING維持）  |
| **滞留回収（DLQ相当）** | `rescue_stuck_videos`: 30分以上 PENDING/PROCESSING を10分毎に再投入 |
| タイムアウト            | ハード15分/ソフト12分。`acks_late` + `prefetch=1` で取りこぼし防止  |
| 定期バッチ              | Celery beat: 滞留回収(10分毎)・保存期間バッチ(日次)                 |

### 残タスク（SLA運用）

- [ ] SLA定義: 「アップロード→分析完了 p95 15分以内」を目標に計測
- [ ] メトリクス: キュー滞留数・処理時間・失敗率を可視化（CloudWatch/Grafana）
- [ ] アラート: FAILED急増・滞留閾値超過で通知（Slack/PagerDuty）

## G#47 固定カメラ/機材連携（自動アップロード）— 設計

- **狙い**: クラブ常設カメラ（VEO等）から練習動画を自動取込し、撮影の手間をゼロに。
- **設計**: 取込用 API キー（デバイス単位）＋ S3 直接 PUT（既存 presigned URL 基盤を流用）
  → 動画到着で自動 `analyze_video` 投入。デバイス登録は Enterprise 機能（`Feature.API_ACCESS`）。
- **着手トリガー**: パイロット(F#41)で常設カメラ保有クラブが2件以上。

## G#48 多競技展開（野球・バスケ横展開）— 設計

- **土台は既にある**: `AthleteProfile.sport` フィールド・検索フィルタは実装済み。
- **競技追加に必要な差分**（サッカーで確立したパターンを競技パックとして複製）:
  1. スコア定義（例: 野球=スイング速度/送球/守備反応/走塁）
  2. ポジション重み（`position_weights.py` の競技版）
  3. プロ水準リファレンス（`pro_reference.py` の競技版）
  4. 分析エンジンの動作認識モデル（競技別・最大の投資点）
- **方針**: `sport` をキーに設定を切替えるレジストリ構造へ段階移行。UIは共通。
- **着手トリガー**: サッカーで有償10クラブ達成後（PMF確認後）。

## G#49 保護者向けアプリ面（継続率UP）— 設計

- **狙い**: 未成年選手の課金決定者は保護者。保護者の納得感が継続率を決める。
- **提供画面**（既存APIで大半実装可能）:
  - 成長サマリ（スコア推移 B#20・練習ログ）
  - 閲覧通知「スカウトがカルテを見ました」（C#30 ProfileViewLog を通知連携）
  - 同意管理（D#32 保護者同意の状態確認・取消）
  - 安全性ダッシュボード（保存期間・削除請求 D#33/35）
- **着手トリガー**: 選手アカウント500件 or 解約理由に「保護者の不安」が出現した時。

## G#50 多言語対応（海外スカウト・アジア展開）— 設計

- **現状**: UI/コメントとも日本語。スコア数値・チャートは言語非依存。
- **段階計画**:
  1. **Phase 1（英語）**: Web を next-i18next 化、バックエンドのコメント文を
     メッセージキー化（`comment_key` + パラメータを返し、フロントで翻訳）
  2. **Phase 2（アジア）**: タイ/ベトナム/インドネシア語。育成年代の裾野が広く
     欧州スカウトの注目市場＝「日本発・アジアの才能DB」ポジションを狙う
- **設計上の注意**: `deep_analysis` 等のコメントは文字列直書きになっているため、
  多言語化時に key 化リファクタが必要（新規コメント追加時は key 化を意識）。
- **着手トリガー**: 海外スカウト/クラブからの問い合わせ、または国内シェア確立後。
