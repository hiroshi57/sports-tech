## 1. なぜ認証が必要か

- 自治体・教育委員会・協会の入札/調達では **Pマーク or ISMS(ISO27001) が事実上の必須要件**。
- 未成年の動画・身体データを扱う本サービスは、認証がないと商談の土俵に乗れないケースが多い。
- Enterprise（請求書払い・一括導入 F#45）の成約率に直結する投資。

## 2. どちらを取るか（方針）

| 認証    | 対象         | 強み                                 | 費用目安(取得) | 期間目安  |
| ------- | ------------ | ------------------------------------ | -------------- | --------- |
| Pマーク | 個人情報保護 | 国内の教育・自治体で知名度が高い     | 約60〜120万円  | 6〜12ヶ月 |
| ISMS    | 情報資産全般 | 国際規格・技術企業として汎用性が高い | 約100〜200万円 | 6〜12ヶ月 |

**推奨**: まず **ISMS(ISO27001)** → 需要に応じて Pマーク追加。
理由: 動画・スコア等「個人情報以外の情報資産」も扱うため ISMS の方が守備範囲が合う。
教育委員会案件で Pマーク指定が多ければ並行取得に切替。

## 3. 取得ロードマップ（ISMS想定・約9ヶ月）

| フェーズ | 期間    | 実施内容                                                       |
| -------- | ------- | -------------------------------------------------------------- |
| 準備     | 1-2ヶ月 | 適用範囲定義・リスクアセスメント・情報資産台帳の作成           |
| 構築     | 2-4ヶ月 | ISMS基本方針・規程類の整備、アクセス制御/ログ/バックアップ運用 |
| 運用     | 2-3ヶ月 | 規程に沿った運用実績づくり・内部監査・マネジメントレビュー     |
| 審査     | 1-2ヶ月 | 一次審査（文書）→二次審査（運用）→認証取得                     |

## 4. 既に実装済みの土台（審査で提示できるもの）

- データ削除・エクスポート API（開示請求対応, D#33）
- 動画の保存期間・自動削除ポリシー（D#35）
- 保護者同意フロー要件（D#32）・プライバシーポリシードラフト（D#31）
- 閲覧監査ログ（C#30 ProfileViewLog）・誤判定補正の運用記録（A#9）
- ロールベースアクセス制御（athlete/scout/coach）・JWT認証

## 5. 残ギャップ（構築フェーズで整備）

- [ ] 情報セキュリティ基本方針・規程類の文書化（組織的対策）
- [ ] 情報資産台帳・リスクアセスメントの実施
- [ ] アクセスログの保全・定期レビュー運用（技術は有り、運用ルール化が未）
- [ ] 委託先管理（AWS/Vercel/Stripe 等の外部サービス評価）
- [ ] 教育・訓練（従業者向けセキュリティ研修）
- [ ] インシデント対応手順・事業継続計画（BCP）

## 6. 体制・予算（概算）

- 推進体制: 情報セキュリティ責任者1名＋事務局（コンサル併用を推奨）
- 予算: コンサル費 50〜100万円＋審査費 50〜100万円＋維持審査 年30〜50万円
- 完了目標: 外販本格展開（Enterprise商談開始）の前まで
