## 1. 料金プラン設計（E#36）

### 課金モデル

**ハイブリッド課金**: 「選手数（席）課金」＋「分析本数の従量」。

- スカウト/クラブは「抱える選手数」と「分析頻度」でコストが決まる → 両軸で価格が伸びる設計。
- 基本枠を月額に含め、超過分だけ従量（overage）で取りこぼしを防ぐ。
- 大口（協会・自治体・大規模クラブ）は無制限＋**請求書払い**の Enterprise で個別見積。

### プラン表

| プラン         | 月額(税抜) | 選手数 | 分析/月 | 超過単価 | 支払い        | 主な解放機能                               |
| -------------- | ---------- | ------ | ------- | -------- | ------------- | ------------------------------------------ |
| **Free**       | ¥0         | 3      | 3       | 不可     | ―             | 基本スコア・レーダー                       |
| **Starter**    | ¥9,800     | 20     | 30      | ¥300/本  | カード        | ＋履歴・比較・ウォッチリスト・レポート出力 |
| **Pro**        | ¥49,800    | 100    | 200     | ¥250/本  | カード/請求書 | ＋成長予測・新着アラート・優先サポート     |
| **Enterprise** | 個別見積   | 無制限 | 無制限  | 見積     | 請求書        | ＋API連携・SSO・個別サポート               |

> 数値の正本は `billing_plans.PLANS`。機能フラグは `Feature` enum。

### 価格設定の根拠

- **Starter ¥9,800**: 個人スカウト/小規模スクールの「趣味〜副業」レンジ。競合の入門帯（月1万円前後）に合わせる。
- **Pro ¥49,800**: クラブ/スカウト会社の本格運用帯。選手100人・分析200本で「1選手あたり ¥498/月」。
  スカウト1人の月間工数削減効果（動画レビュー1本あたり15〜30分 × 200本）に対して十分割安。
- **Enterprise**: 協会・自治体は予算年度・人数規模で大きく振れるため定価を置かず商談。請求書払い必須。

## 2. フリーミアム設計（E#38）

### 無料枠の狙い

Free は「PRO への転換点（conversion point）」を作るための入口。

- **量の制限**: 選手3人 / 分析3本/月。**超過は overage 不可**＝上位プランへ誘導（`allows_overage()=False`）。
- **機能の制限**: 基本スコア・レーダーのみ。比較/レポート/成長予測/アラートは有料で解放。
- 「価値は見えるが、業務では足りない」状態を意図的に作る。

### 転換トリガー（有料化が起きる瞬間）

| トリガー                         | 誘導先   | 実装                                                |
| -------------------------------- | -------- | --------------------------------------------------- |
| 4人目の選手を登録しようとした    | Starter+ | `check_can_add_athlete` → `QuotaExceededError`      |
| 当月4本目の分析を投げた          | Starter+ | `check_can_analyze` → HTTP 402 Payment Required     |
| 複数選手を比較したい             | Starter+ | `has_feature(COMPARE)` ゲート                       |
| 成長予測・新着アラートを使いたい | Pro      | `has_feature(GROWTH_PREDICTION/SAVED_SEARCH_ALERT)` |

### エンフォースメント実装

- 分析枠: `POST /api/videos/{id}/complete` で `check_can_analyze` → 超過は **402**、成功時 `record_analysis` で当期カウント。
- 当期は暦月（月初 UTC でリセット、`_reset_if_new_period`）。
- 未契約ユーザーは自動的に Free サブスクリプションを生成（`get_or_create_subscription`）。

## 3. 課金基盤（E#37）

- **Stripe サブスクリプション**（カード）: `POST /api/billing/checkout` で Checkout Session を作成。
  `STRIPE_SECRET_KEY` 未設定/未導入時は **manual モード**（請求書払い導線）にフォールバックし、環境非依存で動作。
- **請求書払い（B2B）**: `POST /api/billing/invoice` で Pro/Enterprise を `billing_type=invoice` / `status=trialing`（利用開始・月末締め請求）で登録。
- **Webhook**: `POST /api/billing/webhook` で `checkout.session.completed` → プラン有効化、`customer.subscription.deleted` → 解約。
- 本番導入時に必要な残作業: Stripe SDK 依存追加、`STRIPE_WEBHOOK_SECRET` による署名検証、Price ID の事前作成、与信/請求書発行の承認フロー。

## 4. 動画1本あたり原価試算 → 粗利（E#39）

### 前提

- リージョン: AWS 東京（ap-northeast-1）。
- 想定動画: 1本 = 約720p・5分・約300MB。
- 分析パイプライン: 姿勢推定（CPU/GPU）＋スコアリング＋保存。GPU 使用を前提に上振れケースも試算。

### 1本あたり変動費

| 項目                        | 単価根拠                            | 1本あたり概算   |
| --------------------------- | ----------------------------------- | --------------- |
| S3 保存（90日保持）         | ¥3/GB・月 × 0.3GB × 3ヶ月           | ¥2.7            |
| データ転送（アップ/ダウン） | 受信無料＋配信 ¥15/GB × 0.3GB × 2回 | ¥9.0            |
| 姿勢推定 計算（CPUケース）  | Fargate 2vCPU/4GB × 6分 ≒ ¥6        | ¥6.0            |
| 姿勢推定 計算（GPUケース）  | g4dn.xlarge ¥90/h × 2分 ≒ ¥3        | （上振れ ¥3〜） |
| キュー/DB/その他            | Redis・RDS・ログの按分              | ¥3.0            |
| **合計（CPU想定）**         |                                     | **約 ¥21/本**   |
| **合計（GPU上振れ）**       |                                     | **約 ¥24/本**   |

> 保守的に **原価 ¥30/本** として粗利ラインを引く（監視・障害対応・再処理の余裕込み）。

### 粗利（各プランの分析枠を使い切った最悪ケース）

| プラン  | 月額    | 基本枠 | 変動費(¥30/本) | 粗利    | 粗利率 |
| ------- | ------- | ------ | -------------- | ------- | ------ |
| Starter | ¥9,800  | 30本   | ¥900           | ¥8,900  | 91%    |
| Pro     | ¥49,800 | 200本  | ¥6,000         | ¥43,800 | 88%    |

- overage 単価（Starter ¥300 / Pro ¥250）は原価 ¥30 に対し **粗利率 90%超** → 従量でも利益が出る。
- 固定費（人件費・営業・共通インフラ最低額）は本試算に含めず。**限界利益は健全**で、
  損益分岐は「有料アカウント数 × 平均粗利 > 月間固定費」で管理する。

### 示唆（総評）

1. 変動費は月額に対して十分小さく、**価格の主因は原価ではなく提供価値**（スカウト工数削減・見逃し防止）。
   → 値付けは Value-based で妥当。原価起点の値下げ圧力は小さい。
2. リスクは **GPU 単価と再処理率**。撮影品質が低いと再分析が増え原価が膨らむため、
   撮影ガイドライン（F#44）と失敗時の人手補正ループ（A#9）が原価管理に直結する。
3. Free の無料枠（3本/月・原価 ¥90/人）は **獲得コストとして許容範囲**。
   転換率が読めれば CAC 回収期間を Pro 粗利 ¥43,800/月 で割って明確化できる。
